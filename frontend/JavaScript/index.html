# 函数预编译

（函数）预编译的四部曲：

1. 创建 AO 对象 Activation Object(执行期上下文，作用是理解的作用域，函数产生
的执行空间库)
2. 找形参和变量声明，将变量和形参名作为 AO 属性名，值为 undefined
相当于 AO{
 a : undefined,
 b : undefined
}
3. 将实参值和形参统一（把实参值传到形参里）
4. 在函数体里面找函数声明，值赋予函数体
  （先看自己的 AO，再看全局的 GO）

全局的预编译三部曲：

  1. 生成了一个GO 的对象 Global Object（window 就是 GO）
  2. 找形参和变量声明，将变量和形参名作为GO属性名，值为undefined

TIP

预编译完毕 -> 开始执行语句
if里面的变量在执行之前会先进行提升

# 函数参数默认值

如果传入undefined，将触发该参数等于默认值，null则没有这个效果

function foo({x, y = 5}) {
  console.log(x, y);
}

//babel
"use strict";

function foo(_ref) {
  var x = _ref.x,
      _ref$y = _ref.y,
      y = _ref$y === void 0 ? 5 : _ref$y;
  console.log(x, y);
}
如果不传参则会报错

function foo({x, y = 5}={}) {
  console.log(x, y);
}

//babel
"use strict";
function foo() {
  var _ref = arguments.length > 0 && arguments[0] !== undefined ? arguments[0] : {},
      x = _ref.x,
      _ref$y = _ref.y,
      y = _ref$y === void 0 ? 5 : _ref$y;

  console.log(x, y);
}
先判断是否有 arguments 且 arguments.length > 0 && arguments[0] !== undefined
没有则为{}
有了解构默认值则不会报错

# 函数的length属性

  • 指定了默认值以后，函数的length属性，将返回没有指定默认值的参数个数。也就是说，指定了默认值后，length属性将失真。
  • 如果设置了默认值的参数不是尾参数，那么length属性也不再计入后面的参数了,只计算前面的
  • rest 参数也不会计入length属性。

# 参数作用域

  • 如果设置参数x 则函数内部访问x变量则为参数x。倘若没有默认值(且没有实参)则去全局找(GO)
  • 一旦设置了参数的默认值，函数进行声明初始化时，参数会形成一个单独的作用域
  • 这种语法行为，在不设置参数默认值时，是不会出现的。
  //没有设置参数默认值时
    var x = 1;
    function fn (x, y) {
      console.log(x);
      console.log(y);
    }
    fn(); 
    //undefined undefined 
    //有形参x 但是没有赋值，则为undefined
    var x = 1;
    function fn (x, y=x) {
      console.log(x);
      console.log(y);
    }
    fn(10)// 10 10 

  let x = 1;

  function f(y = x) {
    let x = 2;
    console.log(y);
  }

  f() // 1

x没有定义在形参这个作用域，则会去GO找x，x指向GO，函数调用时，函数体内部的局部变量x影响不到默认值变量x。如果GO没有定义x则会直接报错

var x = 1;
function foo(x = x) {
  // ...
}
foo() // ReferenceError: x is not defined
上面代码中，参数`x = x`形成一个单独作用域。实际执行的是`let x = x`，由于TDZ，直接报错

var x = 1;
function foo(x, y = function() { x = 2; }) {
  var x = 3;
  y();
  console.log(x);
}

foo() // 3
x // 1

# 箭头函数

  • 箭头函数没有this,函数体内的this对象，就是定义时所在的对象
  • 不可以使用new命令
  • 不可以使用arguments对象 可以用 rest 参数代替。
  • 不可以使用yield命令，因此箭头函数不能用作Generator 函数。
  • 没有prototype

# 防抖和节流

防抖函数：

  1. 当事件被触发后，不会立即执行，而是等待n秒
  2. 如果n秒内没有再次触发该函数则去调用这个函数
  3. 在n秒内如果频繁触发该函数，则频繁重新计时，目的是阻值在函数频繁触发
  • ajax请求数据
  • 输入框中频繁的输入内容，搜索或者提交信息；
  • 频繁的点击按钮，触发某个事件；
  • 监听浏览器滚动事件，完成某些特定操作；
  • 用户缩放浏览器的resize事件；
function debounce(fn, time, triggerNow) {
  var t = null,
    res;

  var debounced = function () {
    var _self = this,
      args = arguments;

    if (t) {
      clearTimeout(t);
    }

    if (triggerNow) {
      var exec = !t;
      t = setTimeout(function () {
        t = null;
      }, time);
//间隔 time 设置 t = null; 
// 在time内 t 不等于null ->> !t = false ->exec= false,则不执行fn
//如果t=null;执行fn
      if (exec) {
        res = fn.apply(_self, args);
      }
    } else {
      t = setTimeout(function () {
        res = fn.apply(_self, args);
      }, time);
    }
    return res;
  }

  debounced.remove = function () {
    clearTimeout(t);
    t = null;
  }

  return debounced;
}

节流函数：

  • 事件被触发执行，n秒之内只执行一次，(比如攻速是1s一次平A, 我1s按10次平A,也只会1s只触发一次)
    • 监听页面的滚动
    • 鼠标点击/移动
    • 游戏设计
  function throttle (fn, delay) {
    var t = null,
        begin = new Date().getTime();

    return function () {
        var content = this,
            args = arguments,
            cur = new Date().getTime();

    clearTimeout(t);

    if(cur - begin >= delay){
        fn.apply(content, args);
        begin = cur;
    }else{
        t = setTimeout(function () {
            fn.apply(content, args)
        },delay)
    }
    }
 }
