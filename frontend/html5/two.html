# WEB存储

# H5提供了两个在客户端存储数据的新方法

# localStorage

  1. 没有时间限制的数据存储
  2. 没有过期时间除非手动清除

# sessionStorage

针对一个session 的数据存储

localStorage.clear()
localStorage.removeItem(key)
localStorage.setItem(key, value)
localStorage.getItem(key)
key(index)

# localStorage、sessionStorage、cookie

-cookie有过期时间，localStorage不设置过期时间，除非手动清除 -cookie会随着HTTP header发送到服务器端 localStorage不会发送；安全性高 -cookie每次http请求自动携带保存的数据 localStorage不会自动携带保存的数据

# cookie

Cookie 由 HTTP 协议生成，也主要是供 HTTP 协议使用。

  • 跟踪用户方式：
    1. HTTP headers Referrer来源页面
    2. 客户端的IP地址 根据IP追踪()
    3. 用户登录
    4. 胖URL 为用户生成特定版本的Url
    5. Cookies
  • Cookie是服务器发给客户端的特殊信息，cookie是以文本的方式保存在客户端，每次请求时都带上它()
  • 每次http请求都会自动携带保存的cookies数据
document.cookie = newCookie;//设置cookie
  • 参数
    • path(可选) 如果没有定义，默认为当前文档位置的路径；路径必须为绝对路径
    • domain(可选)
      Domain属性指定浏览器发出 HTTP 请求时，哪些域名要附带这个 Cookie。 例如 'example.com'， '.example.com' (包括所有子域名), 'subdomain.example.com'。如果没有定义，默认为当前文档位置的路径的域名部分 (string或null) -Expires，Max-Age -过期时间为1969-12-31则为关闭浏览器cookie消失(删除) max-age 设置过期时间 expires过期

各个属性的写入注意点如下。

  1. path属性必须为绝对路径，默认为当前路径。
  2. domain属性值必须是当前发送 Cookie 的域名的一部分。比如，当前域名是example.com，就不能将其设为foo.com。该属性默认为当前的一级域名（不含二级域名）。
  3. max-age属性的值为秒数。
  4. expires属性的值为UTC 格式，可以使用Date.prototype.toUTCString()进行日期格式转换。

# session cookie

  • 没有设置过期时间,
  • 保存在内存中
  • 意味着这些cookie 只有在浏览器窗口或者标签页开着的时候会存在(会话)

# 持久性cookies

  • 设置了过期时间
  • 保存在用户的硬盘里面
