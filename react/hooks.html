# React Hooks

# Hooks

# Hook规则

  1. 只在顶层使用Hook
  2. 不要在循环，条件或嵌套函数中调用Hook
  3. 只在react函数中使用Hook
  4. 不要在普通函数JavaScript中使用Hook

# useState()

const [count, setCount] = useState(0);
const [count, setCount] = useState(() => 0);
setCount(count + 10)
setCount((prevCount) => prevCount + 1)
  1. useState来自react，需要从react导入，他是一个hook
  2. 参数：初始化值，不设置为undefined
  3. 返回值：数组，包含两个元素
  • 元素1：当前状态的值
  • 元素2：设置状态的函数
  1. useState参数可以传入一个函数返回一个初始值或者直接写出初始值
  2. setCount也可以为一个函数或者直接操作count 与 setState()同理

# useEffect()

Effect Hook 可以让你在函数组件中执行副作用操作

  1. 默认情况下，它在第一次渲染之后和每次更新之后都会执行。（可控制）
  2. 如果你的 effect 返回一个函数，React 将会在执行清除操作时调用它：
  3. effect 在每次渲染的时候都会执行。这就是为什么 React会在执行当前 effect 之前对上一个 effect 进行清除。
  4. 第二个参数可以传递一个数组，React会比较前一次渲染的数组元素和后一次渲染的数组元素进行比较，如果数组中有多个元素变化或者一个，react也会执行effect
  5. 如果想执行只运行一次的 effect（仅在组件挂载和卸载时执行），第二个参数为一个空数组[],这就是告诉react你的effect不依赖于props或state的任何值，所以他永远都不会重复执行，
  6. effect的执行按照从声明先后的顺序

WARNING

如果你熟悉 React class 的生命周期函数，你可以把 useEffect Hook 看做 componentDidMount，componentDidUpdate 和 componentWillUnmount 这三个函数的组合。

# useContext()

const value = useContext(MyContext);

接收一个 context 对象（React.createContext 的返回值）并返回该context 的当前值。当前的context 值由上层组件中距离当前组件最近的<MyContext.Provider> 的 value prop决定。

# useReducer()

const [state, dispatch] = useReducer(reducer, initialArg, init);
/*--------------*/
const initialState = {count: 0};

function reducer(state, action) {
  switch (action.type) {
    case 'increment':
      return {count: state.count + 1};
    case 'decrement':
      return {count: state.count - 1};
    default:
      throw new Error();
  }
}

function Counter() {
  const [state, dispatch] = useReducer(reducer, initialState);
  return (
    <>
      Count: {state.count}
      <button onClick={() => dispatch({type: 'decrement'})}>-</button>
      <button onClick={() => dispatch({type: 'increment'})}>+</button>
    </>
  );
}

useState的替换方案。

  1. 它接收一个形如(state, action) => newState 的reducer，并返回当前的 state 以及与其配套的 dispatch 方法。
  2. 有两种不同初始化 useReducer state 的方式，你可以根据使用场景选择其中的一种。将初始 state作为第二个参数传入 useReducer 是最简单的方法：
  const [state, dispatch] = useReducer(
  reducer,
  {count: initialCount}
);
//React 不使用 state = initialState
  1. 惰性初始化 第三个参数init函数...

# useMemo()

返回一个 memoized 值。

  1. 可以用useMemo()来实现useCallback()
  2. useMemo() 是返回一个memoized的值，而useCallback()是对函数进行优化，则可以使用uesMemo()返回一个函数，达到useCallback()的效果.
const memoizedCallback1 = useCallback(
  () => {
     setCount(count + 1);
  },
  [count],
);


const memoizedCallback2 = useMemo(() => {
  return () => {
    //const [count, setCount] = useState(0);
    setCount(count + 1);

  }
}, [count])

# useRef()

  1. useRef返回一个ref对象。返回的ref对象在组件的整个生命周期保持不变
  2. 主要用来引用dom节点和类组件
  3. 函数组件 ——> forwardRef
  4. 保存数据，返回的ref对象在组件的整个生命周期保持不变
  5. numRef = useRef(count); -> numRef.current
import React,{ useState,useEffect,useRef, forwardRef } from 'react'

const MyInput = forwardRef((props, ref) => {
  return <input ref={ref} type="text"/>
})

export default function App() {
  const [count, setcount] = useState(0);
  const numRef = useRef(count);
  const inputRef = useRef();
  useEffect(() => {
    numRef.current = count;
  },[count])
  return (
    <div>
      <MyInput ref={inputRef}/>
      <button onClick={e => inputRef.current.focus()}>聚焦</button>
      <h2>当前计数qian：{numRef.current}</h2>
      <h2>当前计数hou：{count}</h2>
      <button onClick={e => {setcount((prevState) => prevState + 1)}}>+</button>
      <button onClick={e => {setcount(count - 1)}}>-</button>
    </div>
  )
}

以下是对上述示例发生情况的逐步解释:

  1. 我们通过调用 useRef() 创建了一个React ref 并将其赋值给 ref 变量。
  2. 我们通过指定 ref 为 JSX 属性，将其向下传递给<MyInput ref={inputRef}/>。
  3. React 传递 ref 给 forwardRef 内函数 (props, ref) => ...，作为其第二个参数。
  4. 我们向下转发该 ref 参数到 <input ref={ref} type="text"/>，将其指定为 JSX 属性。
  5. 当 ref 挂载完成，ref.current 将指向 <input> DOM 节点。

WARNING

第二个参数 ref 只在使用 React.forwardRef 定义组件时存在。常规函数和 class 组件不接收 ref 参数，且 props 中也不存在 ref。

Ref 转发不仅限于 DOM 组件，你也可以转发 refs 到 class 组件实例中。

# useImperativeHandle

useImperativeHandle 可以让你在使用 ref 时自定义暴露给父组件的实例值。

  1. useImperativeHandle第二个参数要求返回一个对象。就是对外暴露的inputRef.current,在去重写focus？、？、？
  2. 在自己组件里面再去创建一个ref，进行执行操作
  3. 第三个参数依赖更新
import React,{ useRef, forwardRef, useImperativeHandle } from 'react'

const MyInput = forwardRef((props, ref) => {
  const inputRef = useRef();
useImperativeHandle(ref, () => ({
  focus: () => {
    inputRef.current.focus();
  }
}), [inputRef.current])

  return <input ref={inputRef} type="text"/>
})

export default function App() {
  const inputRef = useRef();
  return (
    <div>
      <MyInput ref={inputRef}/>
      <button onClick={e => inputRef.current.focus()}>聚焦</button>
    </div>
  )
}

# useLayoutEffect()

其函数签名与 useEffect 相同，但它会在所有的 DOM 变更之后同步调用 effect。可以使用它来读取 DOM 布局并同步触发重渲染。在浏览器执行绘制之前，useLayoutEffect 内部的更新计划将被同步刷新。 尽可能使用标准的 useEffect 以避免阻塞视觉更新
