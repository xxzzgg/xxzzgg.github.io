# 规范

  1. 文件夹文件名统一小写 多个单词用-连接
  2. JS变量名称采用小驼峰，常量大写字母 组件采用大驼峰
  3. CSS采用普通CSS和styled-component结合来编写（全局采用普通）
  4. 使用hooks
  5. purecomponent -> memo
  6. 组件内部状态使用useState，useReducer，业务数据全部放在redux中管理
  7. 函数组件内部

  2. 异步请求代码使用redux-thunk

# 数据可变性的问题

尽量保证数据的不可变性 引用值的问题,可以使用浅拷贝解决（...运算符） ImmutableJS

# redux数据处理

reducer函数生成一个新的state 每次都进行...运算符进行浅拷贝所有的数据都需要进行一次拷贝。

# ImmutableJS

npm install immutable

  1. 用一种数据结构来保存数据
  2. 当数据被修改是，会返回来一个新的对象，但是新的对象会尽可能的利用之前的数据解构而不会对内存造成浪费；
