# 小程序

# 与网页开发的区别

  1. 在小程序中 页面渲染和业务逻辑是分开的，分别运行在不同的线程
  2. 无法使用DOM API和BOM API
  3. 逻辑层和渲染层
  4. 小程序主要包含一个描述整体程序的app和多个描述各自页面的page
  5. app有3个部分 app.js(逻辑)app.json(全局配置)app.wxss(全局)

# 页面配置

每一个小程序页面也可以使用 .json文件来对本页面的窗口表现进行配置。页面中配置项在当前页面会覆盖 app.json 的 window 中相同的配置项

# 自适应尺寸单位rpx

  1. rpx：可以根据屏幕宽度自适应，规定屏幕宽为750rpx
  2. 如在iphone6上 屏幕宽度为375px；则750rpx=375px=750物理像素 1rpx=.5px=1物理像素

# 事件分类

事件分为冒泡事件和非冒泡事件：

  1. 冒泡事件：当一个组件上的事件被触发后，该事件会向父节点传递。
  2. 非冒泡事件：当一个组件上的事件被触发后，该事件不会向父节点传递。

# 普通事件绑定

<view bindtap="handleTap">
    Click here!
</view>
-----
<view bind:tap="handleTap">
    Click here!
</view>
//除 bind 外，也可以用 catch 来绑定事件。与 bind 不同， catch 会阻止事件向上冒泡。
